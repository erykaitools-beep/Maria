MEMORY SYSTEMS
  ├─ Episodic Memory (file-based, /memory/*)
  │  └─ Full episodes: input → reasoning → output
  │
  └─ Semantic Graph (in-memory, queryable)
     ├─ Nodes: entities, concepts, facts
     ├─ Edges: relations with confidence
     ├─ Indices: fast lookup by label/type/relation
     ├─ Queries: BFS/DFS traversal + similarity search
     └─ Consolidation: background merge & prune

BACKGROUND PROCESSES
  ├─ Consolidation (every N cycles)
  │  ├─ Merge similar nodes
  │  ├─ Update importance scores
  │  └─ Prune low-value nodes
  │
  ├─ Conflict Detection
  │  └─ Find contradictory beliefs
  │
  ├─ Synchronization (optional)
  │  └─ Extract facts from episodic → semantic


🔑 KEY PRINCIPLES
═════════════════

1. NO STATELESS THINKING
   Every cognitive process reads from and writes to memory.
   The brain without memory is just random noise.

2. BRAIN ↔ MEMORY COUPLING
   Thinking shapes memory (importance boosting, access tracking).
   Memory shapes thinking (retrieval biases, priming effects).

3. EVERY DECISION LEAVES A TRACE
   All reasoning, conclusions, and actions are logged.
   Complete traceability for debugging and learning.

4. METACOGNITION IS MANDATORY
   System continuously monitors itself:
   - Am I confident or uncertain?
   - Am I stuck in a loop?
   - Do I have contradictory beliefs?
   - Is my reasoning sound?

5. GRACEFUL DEGRADATION
   When uncertain, confused, or detecting errors:
   - Acknowledge uncertainty explicitly
   - Request clarification instead of hallucinating
   - Fall back to conservative strategies
   - Log failure for future learning

⚡ PERFORMANCE CHARACTERISTICS
════════════════════════════════

Typical sizes (production):
  • 1,000 nodes, 2,000 edges → < 50ms queries
  • 10,000 nodes, 20,000 edges → 100-200ms queries
  • 100,000 nodes → consider sharding

Memory footprint:
  • ~1KB per node (metadata)
  • ~0.5KB per edge
  • Indices: ~20% of graph size
  • Example: 10K nodes = ~15MB RAM

Consolidation time:
  • 1,000 nodes → ~50ms
  • 10,000 nodes → ~500ms
  • Should run as background process


❓ TROUBLESHOOTING
══════════════════

Common issues:

Graph empty after queries?
  → Check: nodes added before edges!

Query returns nothing?
  → Check: (1) nodes exist? (2) edges connect? (3) max_depth?

Agent crashes on consolidation?
  → Check: not modifying dict while iterating

Memory grows infinitely?
  → Lower pruning threshold or consolidate more often

Slow queries on large graph?
  → Use similarity search instead of BFS
  → Add more indices
  → Consider graph sharding
