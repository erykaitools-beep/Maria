## DEFINITION
Formal logic is the study of valid reasoning through symbolic systems with explicit rules.
It separates the form of an argument from its content, allowing mechanical verification of correctness.

## PART 1: PROPOSITIONAL LOGIC

### Atomic Propositions
A proposition is a statement that is either TRUE or FALSE.
Examples:
  P = "The system is active"
  Q = "Memory is available"
  R = "Task queue is empty"

### Logical Connectives
| Symbol | Name          | Meaning                        |
|--------|---------------|--------------------------------|
| ¬P     | Negation      | NOT P                          |
| P ∧ Q  | Conjunction   | P AND Q                        |
| P ∨ Q  | Disjunction   | P OR Q (inclusive)             |
| P → Q  | Implication   | IF P THEN Q                    |
| P ↔ Q  | Biconditional | P IF AND ONLY IF Q             |

### Truth Table: Implication (P → Q)
P=T, Q=T → T  (valid: premise true, conclusion true)
P=T, Q=F → F  (invalid: promise broken)
P=F, Q=T → T  (vacuously true)
P=F, Q=F → T  (vacuously true)

### Key Tautologies
- Law of excluded middle: P ∨ ¬P (always true)
- Non-contradiction: ¬(P ∧ ¬P) (cannot be both true and false)
- Double negation: ¬¬P ↔ P

## PART 2: INFERENCE RULES

### Modus Ponens (MP) — most important rule
  Premise 1: P → Q
  Premise 2: P
  Conclusion: Q
  
  Example: IF confidence < 0.3 THEN flag_knowledge_gap
           confidence = 0.2
           THEREFORE: flag_knowledge_gap

### Modus Tollens (MT)
  Premise 1: P → Q
  Premise 2: ¬Q
  Conclusion: ¬P
  
  Example: IF system_healthy THEN memory_ok
           memory NOT ok
           THEREFORE: system NOT healthy

### Hypothetical Syllogism (chaining)
  P → Q, Q → R, THEREFORE P → R
  
  Example: low_confidence → learning_needed → Teacher_activated
           THEREFORE: low_confidence → Teacher_activated

### Disjunctive Syllogism
  P ∨ Q, ¬P, THEREFORE Q
  
  Example: (use_fast_path ∨ use_slow_path), NOT use_fast_path
           THEREFORE: use_slow_path

## PART 3: PREDICATE LOGIC

### Predicates and Quantifiers
- Universal quantifier ∀: "for all"   — ∀x: Module(x) → HasState(x)
- Existential quantifier ∃: "there exists" — ∃x: Task(x) ∧ Urgent(x)

### Example in cognitive system context
∀m ∈ Modules: confidence(m) < 0.3 → KNOWLEDGE_GAP(m)
∃t ∈ Topics: confidence(t) = 0.1 → priority_learning(t) = HIGH

## PART 4: LOGICAL CONSISTENCY

### Contradiction Detection
A knowledge base KB is inconsistent if KB ⊢ P and KB ⊢ ¬P for some P.
A cognitive system must check for contradictions when integrating new beliefs.

### Soundness vs Completeness
- Sound: every provable statement is true
- Complete: every true statement is provable
- In practice, cognitive systems aim for soundness; completeness is often undecidable

## PART 5: APPLICATION TO COGNITIVE REASONING

### Decision Logic Pattern
IF precondition_A AND precondition_B THEN action_X ELSE action_Y
This is Modus Ponens applied to decision-making.

### Belief Revision
When new evidence E contradicts current belief B:
  1. Identify the conflict: ¬(E ∧ B)
  2. Assess source reliability: trust(E) vs trust(B)
  3. Retract the less reliable belief
  4. Integrate E into updated knowledge base

### Rule Chaining in M.A.R.I.A. context
K12 uses implicit rule chaining:
  low_confidence(topic) → KNOWLEDGE_GAP(topic) → schedule_learning(topic) → Teacher_session(topic)
This is a valid Hypothetical Syllogism chain.

## SUMMARY
Formal logic provides the structural backbone for valid reasoning.
In cognitive systems, it underpins: decision rules, belief consistency checks,
inference in knowledge bases, and metacognitive self-evaluation.
Understanding formal logic allows M.A.R.I.A. to reason about its own reasoning.
