# META-SYSTEM KNOWLEDGE — Cognitive Learning Material
# Topic: meta-system, self-referential cognition, metacognition
# Confidence target: 0.1 → 0.7+

## DEFINITION
A meta-system is a system that operates on, observes, or controls another system (the object-system).
Meta-system knowledge is the capacity to model, reason about, and modify one's own cognitive processes.

## CORE CONCEPTS

### 1. Levels of Abstraction
- Object level: direct actions and data processing (e.g., parsing input, retrieving memory)
- Meta level: reasoning about those processes (e.g., "am I confident in this answer?")
- Meta-meta level: reasoning about reasoning strategies (e.g., "should I switch inference approach?")

### 2. Self-Model
A cognitive system maintains an internal representation of itself:
  - Current state (active modules, resource load, pending tasks)
  - Capability map (what the system can and cannot do)
  - History of decisions and outcomes
  - Known limitations and knowledge gaps

### 3. Metacognitive Monitoring
The system continuously evaluates:
  - Confidence: how certain is the current output? (0.0 = no basis, 1.0 = fully grounded)
  - Coherence: does the current belief state contradict earlier beliefs?
  - Coverage: are there known unknowns in the active topic domain?
  - Resource efficiency: is the cognitive load proportionate to task value?

### 4. Metacognitive Control
Based on monitoring, the system adjusts:
  - Strategy selection: switch from fast heuristic to slow deliberate reasoning
  - Resource allocation: prioritize modules relevant to current goal
  - Learning triggers: flag low-confidence topics for knowledge acquisition
  - Escalation: defer decisions that exceed self-assessed competence

### 5. Homeostasis in Cognitive Systems
Meta-system knowledge supports homeostasis:
  - Detect deviation from optimal operating state
  - Generate corrective recommendations
  - Verify that corrections produced expected results
  - Maintain a baseline model of "healthy" operation for comparison

## KEY RELATIONSHIPS
- Meta-system ↔ Object-system: bidirectional monitoring and control
- Confidence ↔ Knowledge gaps: low confidence signals where learning is needed
- Self-model accuracy ↔ Decision quality: better self-model → better resource use
- Metacognitive monitoring → Recommendations → Actions → Updated state

## SUMMARY
Meta-system knowledge is the foundation of self-improving cognitive architecture.
Without it, a system cannot know what it does not know, cannot allocate resources intelligently,
and cannot initiate targeted learning. It is the difference between a static program and an adaptive agent.
