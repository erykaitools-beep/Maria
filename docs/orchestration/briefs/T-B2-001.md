# Brief: T-B2-001 — Fix B2 validate handler (file_id dict) + B5 ActionExecutor silent except

## Context

M.A.R.I.A. master audit identified two related production bugs:

**B2 (CRITICAL):** `validate` action 100% fail (9/9 last week) per
`meta_data/action_audit.jsonl`. Codex deep audit (commit `b6b0a35`,
finding DATA-01) root-caused the bug:

- `make_validate_handler()` at `agent_core/routing/handlers/__init__.py:922`

- In production, `file_id` is a **dict** — a full knowledge-index record

all hit this code). Silent swallow explains why `action_audit.jsonl`
records have **no reason field** on failures — the parent caller
believes the recording succeeded.

Both bugs are surgical, sit in adjacent code areas, and bundle well for
one Codex dispatch.

### Subtask 1 — `_normalize_file_id` helper

Add at top of `agent_core/routing/handlers/__init__.py` (or near
`make_validate_handler`, your call):

```python
def _normalize_file_id(value) -> str:
    """Return canonical file_id string.

    Accepts the dict shape produced by the conductor (knowledge-index
    record with 'id'/'file' keys) or a bare string. Raises ValueError
    on unknown shape so the handler can return a meaningful error.
    """
    if isinstance(value, str):
        return value
    if isinstance(value, dict):
        for key in ("id", "file"):
            v = value.get(key)
            if isinstance(v, str) and v:
                return v
        raise ValueError(
            f"file_id dict missing 'id' and 'file' string keys: {value!r}"
        )
    raise ValueError(f"file_id has unsupported type {type(value).__name__}: {value!r}")
```
