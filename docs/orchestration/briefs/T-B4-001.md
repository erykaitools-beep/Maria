## Goal

Add a per-goal cycle counter that increments on every plan executed for
a goal, resets when the plan produces measurable progress, and triggers a
K12 advisory + cooldown when the counter reaches `GOAL_CYCLE_THRESHOLD`
(initial value 5). After this lands, the LEARN→EXAM→REVIEW→LEARN→… loop
that drove 71% abandonment in the 24h test is detected on iteration 5
instead of being indistinguishable from healthy goal churn.

## Locked design decisions (do NOT relitigate)

1. **Threshold:** `GOAL_CYCLE_THRESHOLD = 5`. Module-level constant in
   `planner_core.py` next to the existing threshold constants (~line 54-60).
2. **What counts as "an attempt":** every Plan executed via
   `_finalize_plan` whose `plan.goal_id` is non-empty. Including
   NOOPs only if they carry a `goal_id` (they usually don't — skip
   silently when `goal_id` is empty).
3. **What counts as "progress" (resets counter):**
   - `result.get("success") is True` AND `result.get("skipped") is not True`
   AND at least one of:
   - `chunks_learned > 0` (LEARN)
   - `exams_passed > 0` (EXAM)
   - `reviews_done > 0` (REVIEW)
   - For other action types: `success is True` alone is enough.
   Define a helper `_plan_made_progress(plan, result) -> bool` inside
   `planner_core.py`.
4. **What does NOT count as progress:**
   - `skipped=True` (the B3 fix added this flag — neither failure nor
     progress; counter increments, but treat as "no progress").
   - Any failure (`success=False` and not skipped).
   - NOOP outcomes.
5. **Escalation:** when counter reaches `GOAL_CYCLE_THRESHOLD`:
   a. Apply the existing `STUCK_COOLDOWN_SEC` cooldown:
      `self._state.stuck_cooldowns[goal_id] = time.time() + STUCK_COOLDOWN_SEC`.
   b. Post a bulletin entry of type `EntryType.IMPROVEMENT` with:
      - `topic`: goal description (truncate to 120 chars).
      - `summary`: short Polish text:
        `f"Cel '{goal_desc[:60]}' nie poczynil postepu w {N} probach. K12 powinno przeanalizowac."`
      - `priority`: 0.85 (above ambient, below critical).
      - `metadata`: `{"category": "goal_exhausted_cycle", "goal_id": goal_id, "actions_attempted": N, "threshold": GOAL_CYCLE_THRESHOLD, "action_hint": "self_analyze"}`.
      Use `self._bulletin_store.post(entry)` only if `self._bulletin_store`
      is not None; wrap in `try/except Exception as exc:
      logger.warning("…", exc)` (don't crash planner on bulletin failure).
   c. `logger.warning("[Planner] Goal %s exhausted after %d cycles without
      progress — posted IMPROVEMENT bulletin", goal_id[:12], N)`.
   d. Reset counter for this goal: `self._state.actions_since_progress.pop(goal_id, None)`.
   e. **DO NOT auto-abandon the goal**. Cooldown + K12 bulletin is the
      whole intervention. K12 self-analysis will pick up the IMPROVEMENT
      entry on its next cycle and propose a structured response.
6. **Goal completion / abandonment resets counter:** if `plan.status ==
   PlanStatus.COMPLETED` AND goal status changed to ACHIEVED/ABANDONED
   between before and after execute, drop the counter. (For v1: simpler
   — always drop counter when `_plan_made_progress` returns True. The
   ACHIEVED/ABANDONED case is naturally covered because those plans
   succeed.)
7. **Persistence:** counter dict goes through `to_dict`/`from_dict`.
   Use plain `Dict[str, int]`. Default factory `dict`.
8. **No Telegram notification in v1.** K12 will surface to operator via
   its normal escalation path. Telegram-from-planner stays only for
   existing `_handle_stuck` Level 6 escalation.

```python
def _plan_made_progress(self, plan: Plan, result: Dict[str, Any]) -> bool:
    """T-B4-001: did this plan execution move its goal forward?

    Progress = success AND not skipped AND (action-type-specific signal
    is positive). Resetting the cycle counter on progress prevents false
    K12 escalations on goals that are slowly accumulating chunks/exams.
    """
    if not result.get("success"):
        return False
    if result.get("skipped"):
        return False
    action = plan.action_type
    if action == ActionType.LEARN:
        return result.get("chunks_learned", 0) > 0
    if action == ActionType.EXAM:
        return result.get("exams_passed", 0) > 0
    if action == ActionType.REVIEW:
        return result.get("reviews_done", 0) > 0
    # Other productive actions: success alone is enough.
    return True
```

### Subtask 3 — Add tracker

Add as instance method on the planner class (same class that contains
`_track_nonproductive_repeat`):

```python
def _track_goal_cycle(self, plan: Plan, result: Dict[str, Any]) -> None:
    """T-B4-001: increment cycle counter per goal; escalate at threshold.

    Complements _track_nonproductive_repeat: where that one detects a
    single reflection action repeating ≥20 times, this one detects any
    productive action cycling ≥GOAL_CYCLE_THRESHOLD times on a goal
    without measurable progress. Pattern observed in 24h test
    postmortem: LEARN→EXAM→REVIEW→LEARN→… on goal-meta-learn for 794
    abandoned goals.
    """
    goal_id = plan.goal_id
    if not goal_id:
        return

    if self._plan_made_progress(plan, result):
        self._state.actions_since_progress.pop(goal_id, None)
        return

    count = self._state.actions_since_progress.get(goal_id, 0) + 1
    self._state.actions_since_progress[goal_id] = count

    if count >= GOAL_CYCLE_THRESHOLD:
        self._escalate_goal_cycle(goal_id, count)
        self._state.actions_since_progress.pop(goal_id, None)

def _escalate_goal_cycle(self, goal_id: str, count: int) -> None:
    """T-B4-001: cooldown + K12 IMPROVEMENT bulletin for exhausted cycle."""
    self._state.stuck_cooldowns[goal_id] = time.time() + STUCK_COOLDOWN_SEC

    goal_desc = ""
    if self._goal_store:
        try:
            goal = self._goal_store.get(goal_id)
            if goal:
                goal_desc = (goal.description or "")[:120]
        except Exception as exc:
            logger.debug("[Planner] goal lookup for cycle escalation: %s", exc)

    if self._bulletin_store:
        try:
            from agent_core.bulletin.bulletin_model import (
                BulletinEntry, EntryType,
            )
            entry = BulletinEntry(
                topic=goal_desc or goal_id[:24],
                summary=(
                    f"Cel '{(goal_desc or goal_id[:24])[:60]}' nie poczynil "
                    f"postepu w {count} probach. K12 powinno przeanalizowac."
                ),
                entry_type=EntryType.IMPROVEMENT,
                priority=0.85,
                metadata={
                    "category": "goal_exhausted_cycle",
                    "goal_id": goal_id,
                    "actions_attempted": count,
                    "threshold": GOAL_CYCLE_THRESHOLD,
                    "action_hint": "self_analyze",
                },
            )
            self._bulletin_store.post(entry)
        except Exception as exc:
            logger.warning(
                "[Planner] failed to post goal_exhausted bulletin: %s", exc,
            )

    logger.warning(
        "[Planner] Goal %s exhausted after %d cycles without progress — "
        "cooldown %d min + K12 advisory",
        goal_id[:12], count, STUCK_COOLDOWN_SEC // 60,
    )
```
