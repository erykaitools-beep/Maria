# Brief: T-IR-002 — Wire IntentRouter into homeostasis init + /do command

## Context
M.A.R.I.A. Faza K Deska #1 (IntentRouter). T-IR-001 (commit 0041842) shipped
the IntentRouter library as an isolated module (agent_core/routing/) with
4 local handlers (weather, time, memory, self_model) and feature flag
INTENT_ROUTER_ENABLED (default false). The library is NOT wired into the
runtime — it is dead code until this task.

Authoritative refs:
- CLAUDE.md (project conventions)

  current /do AND reused inside IntentRouter Step 2)

Current /do flow (homeostasis_module.py:2337-2403):
1. TaskIntentDetector.detect(task) — returns DetectedIntent or None.
2. validate_args + is_tool_allowed.
3. ApprovalQueue.submit() with goal_description="[/do] ...".
4. Telegram notify_effector_approval with /efapprove shortcut.

## Goal
Wire IntentRouter so that when INTENT_ROUTER_ENABLED is true, /do tries
local handlers first (weather/time/memory/self_model → execute immediately,
no approval needed). OpenClaw paths (write/read/fetch/etc.) STILL go through
the existing ApprovalQueue + /efapprove flow unchanged. When the flag is
false, /do behaviour is byte-identical to today.

## Definition of Done
- ctx.intent_router is an IntentRouter instance after homeostasis init
  completes, constructed with the live sensors:
    weather_sensor=ctx.weather_sensor (may be None — handler tolerates),
    time_awareness=ctx.time_awareness or equivalent,
    memory_query=ctx.memory_query,
    self_model=ctx.self_model (or None if not wired in this codebase yet
    — check; if absent, pass None and document in commit message),
    capability_router=ctx.capability_router,
    task_intent_detector=<shared single instance reused with _cmd_do>,
    effector_coordinator=ctx.effector_coordinator,
    enabled=None (so it reads INTENT_ROUTER_ENABLED env var).
- _cmd_do logic when ctx.intent_router exists:
    match = ctx.intent_router.route(args.strip())
    if match.path in ("weather", "time", "memory", "self_model"):
        result = ctx.intent_router.route_and_execute(args.strip())
        return result  # bypass ApprovalQueue entirely
    # else: fall through to current ApprovalQueue path UNCHANGED.
- Flag OFF (INTENT_ROUTER_ENABLED=false or unset) → /do behaviour is
  byte-identical to current main: same return strings, same ApprovalQueue
  submission, same /efapprove shortcut.
- Flag ON, local handler matches (e.g. "/do jaka pogoda w berlinie") →
  return real result string from the handler. Zero ApprovalQueue activity.
- Flag ON, openclaw path (e.g. "/do napisz plik /tmp/x z trescia hi") →
  goes through ApprovalQueue exactly as today.
- New tests cover the three cases above. Test file uses mocks for sensors
  consistent with existing test patterns in
  agent_core/routing/handlers/tests/ and agent_core/tests/test_telegram.py.
- Full agent_core/tests/ suite passes.
